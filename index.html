What we built — one Manifest, four layers
Layer 1
Policy

Program metadata, data contracts, detection rules, workflow, reporting — authored as one versioned aml.yaml.

Layer 2
Generation

The spec generates SQL, DAG stubs, data-quality tests, MRM dossiers, control matrices. Spec drift is a CI failure, not a finding.

Layer 3
Runtime

Detectors fire, cases route, BOI refreshes, tuning lab backtests, examiner pack exports. All deterministic — same spec + same data = same output hashes.

Layer 4
Evidence

Immutable SHA-256 hash-chained decisions.jsonl. Spec / input / output / decision hashes. Replayed by Internal Audit byte-for-byte.

How it works in five stages
1
Author the Manifest

CCO / MLRO writes aml.yaml — data contracts, detection rules, workflow, reporting targets. Git PR, 2LoD review, merge.

2
Generate artifacts

aml validate runs JSON Schema + Pydantic checks. Generators emit SQL, DAG stubs, DQ tests, MRM dossier, control matrix.

3
Run detectors

aml run executes rules against your data source. DuckDB engine, deterministic output, SHA-256 hash of every decision written to ledger.

4
Investigate & close

Analysts open the dashboard — alert, KYC, network, sanctions, prior SARs already assembled. Supervisors route, SLA timers run, STR narrative drafted.

5
Export the exam pack

aml audit-pack --jurisdiction CA-FINTRAC builds the ZIP. Every regulator bundle carries the 7-link lineage chain inline. Auditor replays with aml verify-decisions.
